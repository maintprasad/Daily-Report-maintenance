📋 Struktur Google Sheets
Buat sheet dengan nama DAILY_REPORT dengan kolom berikut:
ID Report | Timestamp | Tanggal | Jenis Laporan | Unit | Area | Mesin | Jenis Pekerjaan | Deskripsi | Akar Masalah | Tindakan | Sparepart | Jam Mulai | Jam Selesai | Durasi | PIC | Status | Downtime | Created By | Last Update
Tambahkan sheet kedua bernama PIC_LIST dengan satu kolom:
Nama PIC
Sheet PIC_LIST digunakan untuk sinkronisasi daftar PIC antar semua pengguna. Pastikan GAS kamu mendukung action getPICs dan managePIC.
🚀 Cara Deploy
  1. Buka Google Sheet → Extensions → Apps Script
  2. Paste seluruh isi file daily-report-api.gs
  3. Ganti YOUR_SPREADSHEET_ID_HERE dengan ID spreadsheet kamu
  4. Jalankan fungsi setupSheet() → grant permission
  5. Deploy → New deployment → Web app
  6. Execute as: Me | Who has access: Anyone
  7. Copy URL /exec
  8. Edit file HTML → ganti nilai API_URL di baris paling atas script
📋 Integrasi Work Order Database
Hubungkan dengan WO System (Prasad Seeds) yang sudah berjalan. Sistem otomatis membaca struktur data dari WO System — tidak perlu mapping kolom manual.
URL API Work Order
Masukkan URL yang sama dengan CFG.apiUrl di file WO System HTML. Sistem akan memanggil ?fn=read untuk mengambil semua data WO beserta EquipDB.
Parameter Auto-Match WO ↔ Daily Report
DR dianggap terkait jika tanggalnya dalam rentang ini dari tanggal WO dibuat
⚠ Belum dikonfigurasi
Struktur data yang dibaca dari WO System:
Field WO: id, title, type, status, priority, equipId, unitId, areaId, techId, techName, createdAt, dueDate, notes, closingNote, actualHours
Status WO: Draft → Open → Assigned → In Progress → Done / Cancelled
WO dianggap selesai jika status = Done.
Equipment/Unit/Area di-resolve otomatis dari EquipDB yang ikut di-pull bersama data WO.
